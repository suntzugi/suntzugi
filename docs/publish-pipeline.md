# Suntzugi Publish Pipeline

## Architecture

Single-folder + frontmatter system. All content lives in `content/` with YAML frontmatter for metadata. No file moving between folders, no middleman manifest. The `sync.js` build step reads frontmatter, bakes everything into `index.html`, and generates `content-index.json` for client-side dynamic rendering.

## Content States

| State | Frontmatter | Behavior |
|-------|------------|----------|
| **Draft** | `status: draft` | Shows under "currently being written" with flower. Click shows popup. |
| **Ready** | `status: ready` + `publish_at: <ISO>` | Shows under "soon to be released" with countdown. Click shows popup. Auto-transitions when countdown hits 0. |
| **Published** | `status: published` + `published_at: <ISO>` | Shows under "published". Click opens essay in reading mode. |

## Folder Structure

```
content/                    ← things with identity + lifecycle
  cards/

  text/
    amor.txt
    paciencia.txt
  images/
  audio/
    bgm/
  chraist/

site.yaml                   ← site config (links, display order)
content-index.json          ← auto-generated content index (git-tracked)
scripts/
  sync.js                   ← build step: content → HTML + content-index.json
  auto-publish.js            ← updates frontmatter in-place
  watch.js                   ← file watcher for live editing
```

## Frontmatter Schema

### Cards

```yaml
---
title: Sun
type: card
status: published
---
[card body — uses **Field:** format]
```

### Writing (essays, poems, etc.)

```yaml
---
title: My Time Has Come
type: essay
status: draft
tags: [philosophy, alignment, mettalignment]
date: 2026-03-01
collection:
publish_at: 2026-03-16T05:55:00-08:00
published_at:
---
[body text]
```

- `type`: content type — `essay`, `poem`, `paper`, etc.
- `status`: `draft`, `ready`, or `published`
- `tags`: inline YAML array of topic tags (used for filtering)
- `date`: creation/authoring date (YYYY-MM-DD)
- `collection`: optional grouping label (no UI yet)
- `publish_at`: ISO 8601 timestamp — required for `status: ready`
- `published_at`: set automatically by `auto-publish.js` when publishing

## content-index.json

Auto-generated by `sync.js`, git-tracked via pre-commit hook. Contains all content metadata for client-side dynamic rendering:

```json
{
  "generated": "ISO timestamp",
  "items": [
    {
      "slug": "my-time-has-come",
      "title": "My Time Has Come",
      "type": "essay",
      "status": "draft",
      "date": "2026-03-01",
      "tags": ["philosophy", "alignment", "mettalignment"],
      "collection": "",
      "publish_at": "2026-03-16T05:55:00-08:00",
      "published_at": "",
      "excerpt": "First 200 chars of plain text...",
      "links": ["other-slug"]
    }
  ]
}
```

The client fetches this on page load to render the reading section dynamically with tag filtering. On `file://` protocol (no-JS/fetch failure), the baked HTML fallback remains visible.

## Wiki-links

Use Obsidian-style wiki-links in content body text:

- `[[slug]]` — renders as link with slug as text
- `[[slug|Display Text]]` — renders as link with custom display text

Wiki-links are parsed by the client-side `parseMd()` function and rendered as `<a class="wiki-link">` elements. On hover (desktop, 300ms delay) or touch (mobile), a preview panel shows the linked content's title and excerpt. Clicking navigates based on the target's status:

- **Draft** → shows draft popup
- **Ready** → shows ready popup with countdown
- **Published** → opens essay in reading mode

The `links` field in `content-index.json` is extracted from wiki-links in the body, enabling future graph-view features.

## site.yaml

Structured data for links displayed on the site.

```yaml
links:
  - label: Twitter
    url: https://x.com/suntzugi
  - label: Neodore (stealth)
    url: "#neodore"
    countdown: 2026-03-16T05:55:00-08:00
```

## Writing Content

Create `content/writing/my-essay.md` with frontmatter:

```yaml
---
title: My Essay
type: essay
status: draft
tags: [topic1, topic2]
date: 2026-03-07
---
Your essay body here. Paragraphs separated by blank lines.

**Bold text** and *italic text* work. So do [links](https://example.com).

Reference other content with [[other-slug]] or [[other-slug|display text]].
```

## Publishing Workflow

### 1. Write the draft
Create `content/writing/my-essay.md` with `status: draft`.

### 2. Schedule for publishing
When ready, update frontmatter:
```yaml
status: ready
publish_at: 2026-03-16T05:55:00-08:00
```

### 3. Auto-publish (automated)
GitHub Actions workflow (`.github/workflows/scheduled-release.yml`) runs every 15 minutes:
- Runs `scripts/auto-publish.js` — updates frontmatter: `status: published`, adds `published_at`
- Runs `scripts/sync.js` — rebuilds HTML + content-index.json
- Commits and pushes if anything changed

Manual trigger: `gh workflow run "Scheduled Release"`

## Build Step (sync.js)

`node scripts/sync.js` reads all sources and outputs:
- **index.html**: baked card content, essay lists (fallback), links, audio tracks, text lists, timestamp
- **content-index.json**: structured content metadata for client-side rendering

Sources:
- Card content from `content/cards/*.md`
- Writing metadata from `content/writing/*.md` frontmatter
- Links from `site.yaml`
- Audio tracks from `assets/audio/bgm/` filenames
- Text lists from `assets/text/*.txt`

The pre-commit hook runs sync automatically before every commit and stages `content-index.json`.

## Tag Filtering

Tags defined in frontmatter (`tags: [a, b, c]`) appear as clickable pills below the reading section. Clicking a tag filters the content lists to show only items with that tag. Clicking again clears the filter. Tags update colors with theme changes.

## Timestamp Format (ISO 8601)

```
YYYY-MM-DDTHH:MM:SS+/-HH:MM
```

Common timezone offsets:
- **San Francisco (PST):** `-08:00` (winter) / `-07:00` (summer/PDT)
- **Spain (CET):** `+01:00` (winter) / `+02:00` (summer/CEST)
- **UTC:** `+00:00`

## Quick Reference: Slug Rules

The slug is the .md filename (without extension):
- `my-time-has-come.md` → slug: `my-time-has-come`
- Use lowercase, hyphens for spaces, no special characters
